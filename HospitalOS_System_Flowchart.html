💳 Razorpay

Payment Gateway • 4 API Routes
Handles all online payments for invoices and appointment booking. Flow: create-order → widget opens → patient pays → verify-payment validates HMAC-SHA256 signature → payment recorded. Supports Card, UPI, Net Banking, Wallets. Idempotent (duplicate payment IDs rejected).

Env: RAZORPAY_KEY_ID, RAZORPAY_KEY_SECRET

🤖 OpenAI GPT-4o

AI Service • 5 Use Cases
Powers clinical AI features: SOAP note formatting from doctor's raw text, ICD-10 code auto-suggestion from diagnosis text, pre-consult health brief from patient history, AI discharge summary generation, and patient health assessment (public). Temp: 0.3 for deterministic output. Graceful fallback to templates if API fails.

Env: OPENAI_API_KEY

📲 WhatsApp (Meta Graph API)

Notifications • 7 Message Types
Sends automated notifications via Meta Graph API v18: appointment confirmation, lab results ready, discharge notification, queue token assignment, queue position updates, and "your turn" alerts. Non-blocking — logs warning if credentials missing but never crashes the main operation.

Env: WHATSAPP_API_TOKEN, WHATSAPP_PHONE_NUMBER_ID

💊 FDA OpenFDA API

Drug Safety • Drug Interaction Check
Queries api.fda.gov/drug/label.json for drug interaction data before pharmacy dispensing. Called with array of drug names, returns interaction warnings. Results cached for 1 hour. Graceful degradation — returns empty result on API failure, never blocks dispensing.

📧 SMTP / Nodemailer

Email Service • 4+ Templates
Sends templated emails: patient welcome (with portal setup link, 24h validity), prescription notification, admission confirmation, and other transactional emails. Supports custom SMTP or Ethereal (testing). Non-blocking delivery.

Env: SMTP_HOST, SMTP_PORT, SMTP_USER, SMTP_PASS

📄 PDF Generation (@react-pdf/renderer)

Document Service • 5 PDF Types
Generates PDFs server-side using React-PDF: invoice PDFs, discharge summary PDFs, financial report PDFs, lab report PDFs, and prescription PDFs. Each served via dedicated API routes. Includes hospital branding, structured layout, and QR codes for verification.

These are production readiness gaps identified from the codebase analysis. We'll tackle these module by module.
CRITICAL

No Automated Tests

Zero test files found. No unit tests, integration tests, or E2E tests. 140+ server actions with no test coverage. Production-grade requires at least action-level unit tests and critical flow E2E tests.

CRITICAL

No CI/CD Pipeline

No GitHub Actions, no automated linting on PR, no type-checking in CI. Only a partial lint:ci script covering 4 files. Build can ship with type errors and linting violations.

CRITICAL

Debug / Fix Scripts in Root

Multiple debug files in production root: check.js, checkdb.ts, fix-doctor-fees.ts, fix-fees.ts, fix-org-scoping.ts, deep-inspect.ts, see_users.ts, etc. These should be in a scripts/ folder or removed.

CRITICAL

No Error Monitoring / Observability

No Sentry, no structured logging, no APM. Errors silently caught with empty catch {} blocks. No alerting for payment failures, API outages, or critical lab values.

HIGH

No Database Migrations Strategy

Single migration.sql file in root. No versioned migration history. Prisma migrations folder likely exists but schema changes aren't tracked properly for production rollbacks.

HIGH

No Rate Limiting on API Routes

Login rate limiting exists but API routes (Razorpay, PDF generation, lab verification) have no rate limiting. Vulnerable to abuse and DDoS on expensive endpoints.

HIGH

No HIPAA / Data Compliance Layer

Hospital software handling PHI (Protected Health Information) with no encryption at rest, no data retention policies, no consent management, no data export/deletion capabilities (GDPR/DPDPA).

HIGH

No Caching Strategy

Every page load hits the database directly. No Redis, no ISR, no React Query / SWR for client-side caching. Dashboard KPIs, charge catalogs, and department lists should be cached.

HIGH

No Real-time Updates

Queue display, bed matrix, and nursing station rely on page refreshes. No WebSocket / SSE / Supabase Realtime for live updates. Critical for OPD queue display and nursing station.

MEDIUM

No Background Job Processing

Daily charge accrual, email sending, WhatsApp notifications all happen synchronously in request handlers. Need a job queue (BullMQ, Inngest, or Vercel Cron) for reliability.

MEDIUM

No Backup & Disaster Recovery

No automated database backups documented. No point-in-time recovery setup. No disaster recovery plan. Critical for hospital data that cannot be lost.

MEDIUM

Missing Database Indexes

No explicit composite indexes on high-query columns like (organizationId, status), (organizationId, created_at), (patient_id, date). Performance will degrade with data growth.

MEDIUM

No API Documentation

13 API routes with no OpenAPI/Swagger documentation. No documented request/response schemas. Makes integration testing and third-party integration difficult.

MEDIUM

Patient Portal Security Gaps

Legacy JSON session parsing in getPatientSession() (line 178). Hardcoded fallback org ID 'org-avani-default'. Assessment pages are public with no rate limiting.

MEDIUM

No Payroll / Salary Module in HR

Employee model has salary field but no payroll processing, tax calculations, payslip generation, or salary disbursement workflow. HR module incomplete for production.

## 🛠️ Issues diagnosed and resolved

### 1. Doctor's lab order dropdown — tests not appearing
- **Symptom:** User added test via `/lab/tests` master, but it didn't show up in doctor's portal Lab tab.
- **Root cause:** [LabTab.tsx:25-34](app/doctor/components/LabTab.tsx#L25-L34) had 8 test names baked in as literal `<option>` tags. No DB call.
- **Fix:** Added `useEffect` calling `getTestCatalog()` on mount, filter out `is_available === false`, render dynamic options with name + price. Helpful empty-state message points back to the master.

### 2. "Header details in bill not changing"
- **Symptom:** Edit Invoice → Header Details → change Billing Type → Save → reprint shows same "Category: PAYING".
- **Root cause:** [pdf/route.ts:251 (old)](app/api/invoice/[id]/pdf/route.ts) had literal `<p>Category: PAYING</p>`. Save was working in DB; PDF renderer ignored the value.
- **Fix:** Replaced with conditional reading `invoice.billing_patient_type` (override) → `patient.patient_type` (fallback). Now prints `CORPORATE` / `INSURANCE / TPA` / `PAYING`.

### 3. Insurance company name missing from printed bill
- **Symptom:** Client wants insurance company name on the bill — wasn't there even for clearly-tagged insurance patients.
- **Root cause:** [pdf/route.ts:51 (old)](app/api/invoice/[id]/pdf/route.ts) patient include was `{ full_name, patient_id, phone, age, gender, department }`. No corporate/insurance relations fetched.
- **Fix:** Extended include to load `patient.corporate` and latest active `insurance_policies` with provider. Added blue payer block above patient row showing company name, policy number, plan, billing type. Hidden for cash patients.
